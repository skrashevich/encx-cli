# encx-mock

`encx-mock` — локальный HTTP-сервер с состоянием, который реализует используемые
`encli` endpoints Encounter. Он предназначен для разработки клиента,
интеграционных тестов и проверки retry/timeout-логики без запросов к реальному
домену Encounter.

Это не полная реализация Encounter. Сервер не следует открывать в недоверенную
сеть: настоящей аутентификации нет, почти любые credentials принимаются.

## Быстрый старт

Запуск из репозитория:

```sh
go run ./cmd/encx-mock
```

Или установка и запуск бинарника:

```sh
go install github.com/skrashevich/encx-cli/cmd/encx-mock@latest
encx-mock
```

По умолчанию сервер слушает `0.0.0.0:18080`. В другом терминале:

```sh
encli login -domain 127.0.0.1:18080 -http -login demo -password demo
encli game-list -domain 127.0.0.1:18080 -http
encli status -domain 127.0.0.1:18080 -http -game-id 424242
encli send-code -domain 127.0.0.1:18080 -http -game-id 424242 1
```

Можно использовать любой логин и пароль, кроме `fail:fail`: эта пара специально
возвращает ошибку аутентификации. Прогресс хранится в памяти и разделяется между
всеми сессиями с одинаковой парой логин/пароль. Чтобы начать игру сначала,
перезапустите сервер или используйте другую пару.

## Встроенная игра

Без дополнительных параметров сервер создаёт игру `424242`, команду `5150` и
три уровня по четыре сектора.

| Уровень | Коды секторов | Код закрытия всего уровня |
| --- | --- | --- |
| 1 | `1`, `2`, `3`, `4` | `CODE-1` |
| 2 | `5`, `6`, `7`, `8` | `CODE-2` |
| 3 | `9`, `10`, `11`, `12` | `CODE-3` |

Код сектора закрывает один сектор, а `CODE-N` — весь текущий уровень.
Неправильные ответы сохраняются в истории действий. Во встроенном fixture также
есть тестовые бонусы; текст задания бонуса принимается как ответ на него.

## Настройка

| Параметр | Значение по умолчанию | Назначение |
| --- | --- | --- |
| `ENCX_MOCK_ADDR` | `0.0.0.0:18080` | Адрес HTTP-сервера |
| `-scenario PATH` | нет | Загрузить экспорт Encounter `Game scenario.html` |
| `ENCX_MOCK_SCENARIO` | нет | Путь к сценарию, если не задан `-scenario` |
| `ENCX_MOCK_HAR` | встроенные fixtures | Получить формы ответов из HAR |
| `-v`, `--version`, `version` | | Вывести версию бинарника |

Флаг `-scenario` имеет приоритет над `ENCX_MOCK_SCENARIO`.

## Запуск с экспортом сценария

Режим сценария читает HTML-экспорт страницы Encounter `Game scenario`:

```sh
encx-mock -scenario "/path/to/Game scenario.html"

# Эквивалентный вариант:
ENCX_MOCK_SCENARIO="/path/to/Game scenario.html" encx-mock
```

Из экспорта используются название игры, уровни, задания, ответы секторов,
бонусы, подсказки и таймеры автоперехода. Ответы сравниваются без учёта регистра
и незначащих различий в пробелах. Встроенные сокращения `CODE-N` в этом режиме
отключены: принимаются только ответы из экспорта.

Локальные файлы, на которые ссылается экспорт, по возможности встраиваются как
`data:` URL. Отсутствующие файлы перечисляются при запуске.

## Сценарий для App Store Review

В репозитории есть безопасный fixture для проверки iOS-приложения ревьюерами
Apple без доступа к реальному Encounter:

```sh
go run ./cmd/encx-mock -scenario cmd/encx-mock/fixtures/app_store_review_scenario.html
```

Для проверки с физического iPhone сервер должен быть доступен в локальной сети:

```sh
ENCX_MOCK_ADDR=0.0.0.0:18080 \
  encx-mock -scenario /path/to/app_store_review_scenario.html
```

В настройках приложения укажите HTTP endpoint mock-сервера, например
`http://<LAN-IP>:18080`. Можно использовать любой логин и пароль, например
`apple-review` / `apple-review`.

Игровые данные fixture:

| Уровень | Название | Коды |
| --- | --- | --- |
| 1 | `Welcome` | `REVIEW-START` |
| 2 | `Checkpoint` | `MAP-101`, `LOCK-202`; бонус `BONUS-5` |
| 3 | `Finish` | `FINISH-303`; автопереход через 30 секунд |

Дополнительно можно отправить неправильный ответ, чтобы проверить отображение
ошибки, и код `PZDC`, чтобы включить эмуляцию потери сети для текущей пары
логин/пароль.

## Получение fixtures из HAR

`ENCX_MOCK_HAR` заменяет встроенные формы ответов шаблонами, полученными из HAR
при запуске:

```sh
ENCX_MOCK_HAR=/path/to/dump.har encx-mock
```

HAR должен содержать:

- JSON-ответ игрового движка от `POST /gameengines/encounter/play/...`;
- JSON-ответ списка игр от `GET /home/?json=1`.

Сервер сохраняет форму записанных ответов, но заменяет идентификаторы
игры/команды и изменяемые поля прогресса. Исходный HAR только читается при
запуске. Не коммитьте captures с настоящими cookies, токенами или приватным
сценарием.

`ENCX_MOCK_HAR` можно сочетать с режимом сценария: HAR задаёт форму информации
об игре, а сценарий — игровой контент. User details всегда возвращаются из
небольшого очищенного встроенного шаблона.

## Эмуляция потери сети

Отправьте `PZDC` как ответ уровня, чтобы на минуту включить задержку запросов для
текущей пары логин/пароль:

```sh
encli send-code -domain 127.0.0.1:18080 -http -game-id 424242 PZDC
```

В течение этой минуты каждый запрос ждёт до двух секунд без данных. Клиент с
меньшим timeout получает timeout; клиент с большим timeout после двух секунд
может получить пустой ответ. Для немедленного продолжения используйте другую
пару логин/пароль или перезапустите сервер.

## Docker

В контейнере сервер должен слушать все интерфейсы. Файлы сценария и HAR нужно
смонтировать внутрь контейнера:

```sh
docker run --rm -p 18080:18080 \
  -e ENCX_MOCK_ADDR=0.0.0.0:18080 \
  ghcr.io/skrashevich/encx-mock

docker run --rm -p 18080:18080 \
  -e ENCX_MOCK_ADDR=0.0.0.0:18080 \
  -v "$PWD/fixtures:/fixtures:ro" \
  ghcr.io/skrashevich/encx-mock \
  -scenario /fixtures/Game-scenario.html
```

## Реализованное поведение и ограничения

Сервер покрывает логин, user details, список и детали игр, принятие команды,
игровые GET/POST-запросы, статистику, подтверждение взноса и заглушку anti-spam,
которые использует этот репозиторий.

Состояние хранится только в памяти. Доступна одна фиксированная игра и команда;
нет organizer/admin API, постоянного хранилища, TLS и полной эмуляции правил
Encounter. Неизвестные ID игр и маршруты возвращают `404`, а защищённые
endpoints без mock-сессии — `401`.
